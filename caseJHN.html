Overview

A design handed over.
A website shipped.

John had a polished Figma file ready, the visual direction was clear, but it existed only as a static document. The task was to close the gap between design and delivery: turning every layout decision into clean, cross-device frontend code without losing fidelity or feel.

The challenge wasn't just replicating the design, it was building in responsive behaviour the Figma file didn't define, and doing it in a way that felt native to the original vision.

Challenge & Solution

Problem
  • –Design existed only as a static Figma file
  • –No responsive behaviour defined for smaller screens
  • –Interactive states needed to be invented, not copied
  • –Performance and asset optimisation unconsidered
Solution
  • →Converted every Figma frame into semantic HTML
  • →Designed a responsive system across 4 breakpoints
  • →Added transitions true to the design language
  • →Optimised assets, deployed to GitHub Pages

Process

How I worked through it

01
Design audit

Combed through every Figma frame, typography scales, spacing rules, colour tokens, component patterns.

02
Responsive planning

Mapped out how each section adapts across mobile, tablet, and desktop before writing a single line.

03
Build & iterate

Developed section by section, checking against the Figma at every stage and refining spacing and rhythm.

04
Polish & ship

Added micro-interactions, optimised images, tested across devices, deployed to GitHub Pages.
